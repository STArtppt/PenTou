# PenTou 笔头

> **本地优先的 AI 对话管理器**
> 把散落在各平台的 AI 对话，沉淀为可检索、可加工、可流转的本地知识资产。

---

## 一、为什么是 PenTou

如果你日常在 ChatGPT、Claude、DeepSeek、Gemini、Cursor、Copilot、Codex、Claude CLI 之间频繁切换，会很快意识到三个痛点：

1. **对话散落**：每个平台一套历史记录，账号注销、服务下线、平台改版都可能让一段灵感悄无声息地丢失。
2. **难以再加工**：平台原生 UI 是给"聊天"设计的，不是给"知识管理"设计的——没有文件夹、没有跨平台搜索、没法对一段长对话做摘要或重写。
3. **数据不归你**：所谓的"历史记录"放在云上，导不出、改不动、搜不到。

**PenTou（笔头）** 的取名灵感来自"好记性不如烂笔头"——把每一次有价值的 AI 对话固化为本地资产，让 AI 的产出真正可以**沉淀、回查、二次加工**。

PenTou 的核心设计原则只有一条：

> **数据归用户所有，以原生 Markdown 形式存储在本地文件系统中。**

没有专有数据库、没有云服务锁定、没有付费墙——你随时可以用 VSCode、Obsidian、Git 直接消费这些 `.md` 文件。

## 二、当前已实现的能力

### 1. 多源对话导入

PenTou 支持把不同平台、不同形态的 AI 对话归口到同一个本地仓库：

| 来源 | 输入形式 | 处理方式 |
| --- | --- | --- |
| ChatGPT 官方导出 | `conversations.json` | 沿 `mapping` 活跃分支递归还原消息流 |
| DeepSeek 导出 | `conversations.json` | 还原 REQUEST / RESPONSE / THINK；思考链自动转为 Markdown 引用块 |
| CLI 工具日志 | `.jsonl`（Claude CLI、Codex 等） | 逐行解析，识别 `type` / `role` 字段 |
| IDE / 脚本产物 | `.md`（含 frontmatter） | 严格按 `## User` / `## ChatGPT` 等角色边界切分 |
| 平台分享链接 | URL（DeepSeek 等） | 后端双轨抓取：API 嗅探优先，浏览器渲染降级，绕过 CORS 与基础反爬 |

**一次能拖入多个文件**，单个文件解析失败不会中断同批次的其他文件，UI 会精确告诉你哪条出了问题。

### 2. 本地 Markdown 存储

每一条导入或生成的对话，最终都会落盘为：

```
data/conversations/<id>.md
```

文件结构是**人类可读**的 frontmatter + 消息正文：

```markdown
---
id: ...
title: ...
platform: ChatGPT
date: 2026-05-12T...
folderId: ...
---

## User
...

## AI
...
```

应用重启时全量扫描即可恢复状态——**没有索引数据库、没有 schema 演进的负担**。你可以把整个 `data/` 目录用 Git 管起来，跨设备同步、版本回溯都是现成的。

### 3. 三栏沉浸式浏览

- **左侧栏 Sidebar**：对话/文档双 Tab 切换、文件夹树形组织（可拖拽分类、可批量选择）、关键词过滤、明暗主题与中英文一键切换。
- **中央内容区 MainContent**：与平台原生视觉接近的气泡渲染；代码块原生高亮、一键复制；Markdown 完整支持。
- **右侧导航 RightNav**：基于 User 提问自动生成"问题大纲"，点击即可滚动定位；长文本悬停展开完整内容。

### 4. 对话 → 文档：知识沉淀闭环

PenTou 不止是阅读器。它内置了一套**对话 → 文档 → 批注 → 重写**的加工闭环：

| 入口 | 行为 |
| --- | --- |
| **一键转文档** | 调用你配置的 LLM（BYOK，兼容 OpenAI 接口），把整段对话重写为结构化文档；P95 < 20 秒。 |
| **消息摘录** | 不依赖 LLM，直接把对话中的高价值片段按统一模板写入同源文档。 |
| **外部资料导入** | 走 `markitdown` 管线，PDF / Docx / HTML / PPTX 等转 Markdown；未安装也能直接收 `.md`，半装状态会给出明确诊断。 |

### 5. 阅读、批注、AI 重写

文档进入仓库后有两种工作模式：

- **阅读模式**：选中文字即可高亮、加评论。批注作为 sidecar JSON 持久化，刷新 100% 保留。
- **编辑模式**：所见即所得的 Markdown 编辑。

**关键能力是"批注驱动重写"**——你把若干条批注作为修订意见交给 LLM，PenTou 会生成一份新版本，原版本完整入历史。任何一次重写都能在版本面板里找回、对比、回滚。

### 6. 推送到 Obsidian

PenTou 不打算成为你的最终知识库——它擅长**采集与加工**，归档交给 Obsidian。一键推送通过 `obsidian://` URI 唤起目标 Vault；超长内容自动降级走剪贴板。

### 7. 私有化部署

除了本地开发模式（`pnpm dev`，无鉴权），PenTou 提供完整的 Docker 部署方案：

```bash
docker run -d \
  --name pentou \
  -p 127.0.0.1:7766:7766 \
  -e PENTOU_PASSWORD='your-strong-password' \
  -v /srv/pentou/data:/app/data \
  ghcr.io/<owner>/pentou:latest
```

- 同时支持 `linux/amd64` 与 `linux/arm64`（兼容群晖、极空间等 NAS）
- 默认非 root（uid 1000）
- 数据卷与代码彻底解耦，升级 = `docker pull && docker compose up -d`
- TLS / 域名由你自己的反向代理（Caddy / Nginx / Traefik）承担

## 三、技术栈

PenTou 主张**轻量而完整**：

- **前端**：React + Vite + Tailwind CSS（v4），UI 组件遵循 shadcn 约定。
- **后端**：自研 Vite 插件（`pentouServerPlugin` / `documentsPlugin` / `markitdownPlugin` / `obscura`）以 Express 风格中间件接管本地 I/O 与跨域抓取——开发态与生产态同构。
- **渲染**：React Markdown + Framer Motion 微交互。
- **协作规范**：根目录 `CLAUDE.md` / `AGENTS.md` 声明 AI 协作准则（基于 Karpathy 编程规范），所有功能走 `spec.md → 评审 → 编码 → implementation-log.md` 的闭环。

## 四、谁会从 PenTou 受益

- **AI 重度用户**：每天产出大量对话，需要跨平台统一留存与回查。
- **知识工作者 / 内容创作者**：把对话当作素材池，提炼为文章、策划、笔记。
- **开发者**：归档技术方案、问题诊断、代码片段；可直接 Git 化管理。
- **数据隐私敏感者**：拒绝把对话上传到第三方云端，要求一切落在自己掌控的设备上。
- **Obsidian / Markdown 工作流用户**：希望 AI 对话能无缝并入既有知识库。

## 五、后续规划

### Roadmap A：混合检索与全文检索

当前的搜索仅做标题与正文的关键词匹配。下一阶段会引入**混合检索**：

- **全文检索**：对全部对话与文档建立本地索引（BM25 或类 SQLite FTS5），保持本地优先、零云端依赖。
- **语义检索**：可选启用嵌入向量索引，使"我之前是不是和 AI 讨论过 X"这类模糊提问也能命中。
- **混合排序**：关键词 + 语义双路召回再融合，兼顾精确度与召回率。

目标是让 PenTou 从"对话仓库"升级为**真正可被检索的个人 AI 知识库**。

### Roadmap B：右侧 AI 对话侧边栏

在现有三栏布局的右侧，新增一个**常驻的 AI 问答面板**：

- **上下文感知**：默认把当前选中的对话 / 文档作为问答上下文，无需复制粘贴。
- **跨库提问**：结合 Roadmap A 的检索能力，对全量历史发起追问（如"汇总我所有关于 RAG 的讨论"）。
- **生成即沉淀**：问答结果可一键转为新文档，进入既有的批注 / 重写 / 版本 / 推送闭环。

这两条路线最终会汇合：**检索负责"找回过去"，AI 侧边栏负责"基于过去生成未来"**——把 PenTou 从被动的归档工具，变成主动的思考伙伴。

## 六、设计哲学

PenTou 不追求大而全。它在以下几个取舍上态度鲜明：

| 选择 | 不选择 | 原因 |
| --- | --- | --- |
| 本地 Markdown 文件 | 专有数据库 | 数据归属用户，工具链通用 |
| 一对话一文件 | 单一索引文件 | 易备份、易 diff、易迁移 |
| BYOK 自带 Key | 内置模型 / 付费墙 | 用户对成本与隐私完全可控 |
| 自研轻量后端 | 重型框架 | 单一仓库即可端到端运行 |
| Spec 驱动开发 | 直接动手写 | 每个功能可追溯、可评审 |

> **PenTou — 停止碎片化对话，开始构建属于你的 AI 知识资产。**
